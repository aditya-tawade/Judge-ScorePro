# Contributing to Judge Score Pro

Thank you for your interest in contributing to Judge Score Pro! We welcome contributions from the community to help make this the best real-time scoring platform.

## How to Contribute

1.  **Fork the Repository**: Create a fork of the project on GitHub.
2.  **Create a Branch**: Create a descriptive branch name for your changes (e.g., `feature/add-live-polling` or `fix/leaderboard-sorting`).
3.  **Implement Your Changes**: Write clean, documented code and follow the existing style.
4.  **Test Your Changes**: Ensure your changes don't break existing functionality.
5.  **Submit a Pull Request**: Provide a clear description of what your PR does and why it's needed.

## Style Guide

- Use standard JavaScript/React best practices.
- Ensure all UI components are mobile-responsive and follow the glassmorphism aesthetic.
- Use Tailwind CSS for styling.
- Ensure any new API routes have proper error handling.

## Code of Conduct

Please be respectful and professional in all interactions. See [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) for more details.
